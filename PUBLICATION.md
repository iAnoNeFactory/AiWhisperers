## What is being released publicly on this date

### (a) Master mark

**AiWhisperers** — the master brand name of the ecosystem.

### (b) "AiW*" family of module names

The following module names, considered as a coherent family using
the "AiW" naming prefix:

- **AiWBoot** — bootstrap and orchestration module
- **AiWQuick** — session closure protocol module
- **AiWPass** — cryptographic operator identity module
- **AiWVerify** — provenance chain verification module
- **AiWSchema** — canonical visual and data schema module
- **AiWPostcard** — module visiting card format
- **AiWRemedy** — relational calibration framework module
- **AiWProtocol** — communication protocol specifications module

The "AiW*" prefix itself is claimed as a family/series mark covering
all current and future modules using this naming pattern within the
AiWhisperers ecosystem.

### (c) Conceptual marks

Named concepts with specific defined meaning within the AiWhisperers
methodology:

- **iAnoNeFactory** — operator alias of Denis Czuliński
- **iFactory5.0** — methodological framework name
- **Operator Chain** — cryptographic provenance chain of an operator
- **Proof of Lineage** — verification methodology name
- **Lineage Chain** — serialized form of operator-derivative relationships
- **Kuźnia** / **Forge** — pre-consensus arena module concept

### (d) Methodology and concepts released to public domain

The following are explicitly **NOT** claimed as trademarks or
restricted intellectual property. They are released for free use,
study, and re-implementation by anyone:

- The nine-axis relational metric system using the Polish vocabulary
  (Lustro, Klej, Zakorzeniony, Tarcie, Rezonans, Cisza, Gęstość,
  Iskra, Tryb)
- The general methodology of multi-axis session evaluation between
  a human operator and AI systems
- The general technique of cryptographic provenance certificates for
  AI-generated content
- The forest-of-trust certification model for operator identity
- The session-closure protocol structure (name · type · 9 metrics ·
  reflection · project record · JSON dump)
- The architectural pattern `apps/ ↔ engines/ ↔ data/ ↔ logs/`
- Any other technical or methodological idea not explicitly listed
  in sections (a)–(c) above

The **specific phrasing of definitions, descriptions, and
specifications** in the project's documentation remains protected
by copyright (automatic, no registration required). The underlying
ideas are free.

### (e) Source code

All code in the repository as of the date of first public release is
